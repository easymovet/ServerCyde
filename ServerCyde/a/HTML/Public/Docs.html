QuickStart Guide

This step by step guide will get you up and running using the ServerCyde API in no time

1. Create a new ServerCyde "Site"

The script tag that you will put on your page is tailored to have the database call that you create just for your site, so create a new site, add the details and then you will get a script tag to place in your source.

2. Authentication

In order to run most calls the client needs to have am identity cookie. To get one they need to sign up or sign in to your site

Create a form with 4 fields: email, name, password, rememberMe

In the user tab you will see the all the calls that you need to create and manage users, clicking on one of them will give you a test form. That form will show you the required fields and a sample JSON response.If you are building a native mobile app then you can make calls directly to the URLs that you see in the test form instead of using the javascript library

To create a new user wire a function that runs ServerCyde.Users.SignUp to the onlcick event of the submit button and pass in the serialized form or a data object eg: {email:"email@email.com", name: "dude" ... }

Build a sign in form in a similar fashion

Hint: You can just copy the form and functionaly from the Example site

3. Create a Domain

In the domain tab you can see the domains that you already have and you can create new domains

Create the domains that you will be needing for your app

4. Create Modify Calls

To add data you will be passing data into a modify call that you have saved.

You will invoke the call using the nickname that you give it.

If you build a call that has a token that requires session based info like the current user id or current user name then the call will only run if the current user is authenticated and email confirmed

Once you save a modify call you can see the syntax neede to invoke it in the Modify tab

5. Create select calls, emails, and comet features

These work in a similar way to the modify calls, at this point you should have the hang of it.

6. Publish your site

All of the functions use cross domain resources so there is no difference between running the site locally and running on a host, all you have to do is upload your HTML to a web server

Tokens

Whats makes servercyde calls secure is that you can have the server insert values into calls instead of leaving them exposed to browser manipulation. For example if you only wanted to let a user see results from the database that they themselves inserted you would build a query like: Select * From tweets Where UserID = [%currentuserid%] as long you you built the modify to also inject that same value for userid you will be able to ensure secure results.

Comet

The comet feature works like a broadcast channel. Everyone who is subscribed will get updates as soon as anyone publishes something. To prevent spam only autheticated and email confirmed users can publish but anyone can subscribe.

For security reasons you may want to ensure that you don't execute the values that get published or really any user submitted data to prevent undesired code running on recipients browsers.
